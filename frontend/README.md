# Logbook frontend

The frontend is a responsive React dashboard for the Logbook API. Production uses Nginx as a same-origin proxy so the backend API key is injected server-side rather than included in browser JavaScript.

Use the root [README](../README.md) for deployment. This document describes the dashboard and frontend development workflow.

## Stack

- React and TypeScript
- Vite
- Tailwind CSS 4
- Radix-based UI components
- TanStack Query
- Recharts
- React Router
- Lucide icons

## Pages

### Recent activity

Shows the three newest events for each active event type, along with time since the latest start and latest completed finish. Ongoing events do not erase the previous completed finish.

### Events

Provides the complete paginated history with:

- event-type filtering
- case-insensitive note search
- finish, edit, and delete actions
- value, unit, text, note, start, and end corrections
- reopening a duration event by clearing its end

### Event types

Creates and edits the controlled event catalog. Event types may define a measurement dimension, default unit, description, and voice aliases. The logging action—not the event type—determines whether each event is a point or duration event.

### Units

Creates measurement dimensions and units, including aliases and affine conversion factors. Unsafe conversion edits and deletion of referenced units are blocked by the backend.

### Series detail

Displays:

- recorded values in the selected display unit
- optional bucketed sums
- forward-aligned time until the next event start
- event durations
- value-per-duration rates
- local-time presets and custom date ranges
- drag-to-select chart ranges
- an edit-latest action

The recorded-value and time-gap curves can be toggled independently. Continuous timestamp axes preserve the real spacing between events.

All active pages poll their API queries every 10 seconds, including when embedded in a Home Assistant Webpage dashboard. TanStack Query preserves structurally equal results, so the visible page updates only when returned state changes. Preset chart ranges roll forward with each refresh; custom date ranges remain fixed.

## Data behavior

### Units

The browser submits the user's original value and selected unit key. It does not perform canonical conversion. The backend validates compatibility, stores the canonical value, and returns series data in the requested display unit.

### Time

Interactive writes use explicit ISO timestamps generated by the browser. Chart requests include the browser's IANA timezone so daily and weekly buckets align to local midnight.

### Event shapes

- **Log point** creates a zero-duration event.
- **Start duration** creates an ongoing event.
- **Finish** closes an ongoing event and may optionally supply its final value.
- **Edit latest** corrects the newest event without first looking up its UUID.

## Production deployment

The root Compose stack builds and runs the frontend together with the API:

```bash
cd ..
./scripts/deploy-stack.sh
```

The dashboard is available on `FRONTEND_BIND_ADDRESS:FRONTEND_PORT`, normally port `8790`.

Nginx proxies `/api/*` to the backend and injects the bearer token from the container environment. Direct API access remains controlled by the backend's authentication and CORS configuration.

## Add to Home Assistant

Create a **Webpage dashboard** pointing to the frontend URL:

```text
http://<logbook-host>:8790
```

When Home Assistant is served over HTTPS, publish the dashboard over HTTPS as well to avoid browser mixed-content blocking.

## Local development

```bash
npm ci
VITE_DEV_API_TARGET=http://192.168.68.62:8787 \
VITE_DEV_API_KEY=your-api-key \
npm run dev
```

Validation:

```bash
npm run check
```

`npm run check` runs linting, strict type checking, tests, and the production build. Frontend tests cover API headers, event actions and editing, filtering, chart timing, local calendar alignment, aggregation, tooltips, and responsive chart behavior.
